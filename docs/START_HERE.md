# 现在先做什么（先别慌）

你现在只需要做 3 件事：

1. 锁定 MVP 范围（不再加新功能）
2. 按规格做第一个可演示版本（Web）
3. 再决定技术栈细节和客户端扩展

## 当前 MVP（已确认）

1. 新建任务，勾选完成，完成历史保留
2. 任务支持优先级
3. 任务支持 DDL，并在到期前提醒
4. 多端云同步（使用 Token，不用账号密码）

## 推荐顺序（非常简化）

1. 先读 [SPEC-v0.1.md](./SPEC-v0.1.md)
2. 实现 Web 版最小闭环：新增 -> 完成 -> 同步 -> 提醒
3. 每完成一个功能，就更新一次规格文档
4. MVP 稳定后，再做桌面端/Android

## 如果切换 Agent（例如 Copilot -> Codex）

1. 先让新 Agent 阅读仓库根目录的 [AGENTS.md](../AGENTS.md)
2. 再按 AGENTS.md 里的顺序读取 START_HERE、SPEC、DEV-CHECKLIST
3. 让新 Agent 先复述一次“当前里程碑 + 下一步最小动作”再开始改代码

## 今天就能开始的任务

1. 把数据模型和接口先定下来（见规格文档）
2. 写出最小 API（创建任务、更新状态、获取列表、同步）
3. 做一个最简单的 Web 页面验证流程

## 暂时不要做的事

1. 不要先做复杂 Agent 功能
2. 不要先追求多端同时开发
3. 不要先引入太多中间件

## 卡住时怎么做

1. 回到规格文档，把问题写成一句话
2. 只解决当前里程碑的问题
3. 每次只改一个小点，保证可验证
