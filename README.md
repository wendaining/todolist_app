# Todolist App

## Start Here

> 我想要做一个跨平台的TodoList软件，这是我同学给我的一个项目方面的建议，我阐明一下我目前的水平，以及当前的需求，请你和我沟通一下最终我们应该如何去实现。
>
> 项目省流版说明：一个跨平台的私有的Todolist应用程序（Windows, MacOS, Linux, Android)
>
> 我的背景需求：我最近想要搞一点项目，方便写在我简历上，来方便我找后端领域的实习（实际上这个项目应该是全栈的）。
>
> 我个人的技术水平
>
> - 后端语言层面只懂基础的Java语法，但是有考虑这个项目去用Typescript / Golang等别的语言来实现
> - 前端层面完全没手写过，但是知道有清楚一些热门的框架，如Vue, React等
> - 对于计算机复杂一点的特性，如高并发，云原生，计算机网络相关的知识几乎不算有认知
> - 对数据库的认识只有会写一点sql，别的一概不知
> - git可以算是熟练掌握
> - 消息队列也不会
> - 跨平台的部分不了解
> - 我认为这个App应该做好限权和鉴流，但是我又有听说这个项目适合Serverless，但是我不懂这个原理，安全方面的知识我也不了解
> - 剩下没提到的可以默认我不会
>
> 因此，你可以容易发现，我也不会分析这个项目的业务需求，各个组件采用何种设计模式，采用什么技术栈最为合适，所以我打算先和你沟通几轮，来确定这个项目的大体的设计框架文档与Spec。
>
> 我在这个项目中想学会的东西：
>
> 1. 对于整体后端开发的一个整体的认识，能大概学会（有认识）尽可能多的技术栈的使用
> 2. 对于第一条，具体而言，我不熟悉的比如并发编程、数据库、消息队列、网络编程、跨平台等
> 3. 对于Agent的新特性的使用的学习，比如sub-agent, skills等
>
> 我的一些需求：
>
> 1. 我觉得学习覆盖面最大化 > 简历展示深度 > 上线可用产品
> 2. 我个人觉得可以先做Web，但是后面也要进行客户端的开发
> 3. 需要云同步，那么我感觉账号登录这一块也值得说道说道，比如说托管到CloudFlare，账号密码的安全性如何保证什么的
> 4. MVP必须包含的功能：
>     1. 一个todolist，输入之后会出现一个条目，然后点击勾可以标注为完成，但是完成的条目的历史需要保存
>     2. 不同的条目之间需要云同步，不采用账号密码，可以采用一个Token
>     3. 支持设置DDL时间，DDL前给通知
>     4. 可以设置任务的优先级
> 5. 项目周期感觉暂且不明了，我觉得一步一步开发吧
> 6. Agent学习我想都学一下，但是主要是学习如何在软件开发中使用Agent，后续可能考虑把这个项目扩展为一个Agentic项目
> 7. 我希望通过这个项目学会如何使用Agent，包括但不限于基本的skills的开发workflow，以及如何最高效地使用Agent，这一点非常重要，我非常想学习这些东西！
